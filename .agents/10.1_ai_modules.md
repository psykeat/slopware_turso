# Implementation Plan: Zero-Footprint AI Email Classification & Action Planning

This revised plan specifies the implementation of the first wave of AI-powered ERP flows in `slopware` based on the AI V2 proxy architecture, obeying a strict **Zero-Footprint UI** rule. Mail is the first concrete module example, not a separate AI shell architecture:

1. **Mail Classification & Action Suggesting** (Priority 1)
2. **Mail to Document Draft/Order Proposal** (Priority 2)

AI features are entirely invisible by default, only appearing when explicitly invoked by the user, preserving the layout of the mailbox for users and tenants who do not use AI. Review and apply flows are module-specific; the overlay shell stays shared.

---

## User Review Required

> [!IMPORTANT]
> **Zero Layout Footprint**: No reserved inspector panels, empty sidebar blocks, or dedicated layout dividers are introduced in `email.tsx`. The standard workspace remains identical to its current shape.
> **Command-First Invocation**: The AI Assistant is exposed as a toolbar action and registered globally in `CommandProvider` (primary hotkey: `Alt+A`; `F4` remains reserved for archive/delete in the platform keyboard model).
> **Overlay UI Sheet**: On invocation, a shared overlay `Sheet` glides from the right side, housing the interactive, multi-step action plan, dynamic validations, and transactional override forms for the mail module.

## Proposed Changes

### Component 1: Database & Services (`packages/db`)

#### [MODIFY] [ai-discovery.ts](file:///home/ubuntu/slopware/packages/db/src/services/ai-discovery.ts)

- Add `emailThread` and `emailMessage` as semantically discoverable entities.
- Update scopes for `document`, `documentLine`, `address`, `emailThread`, `emailMessage`, and `article` to handle `mail-classification` and `mail-to-document-draft`.
- Add `apply-ai-mail-classification` to `BOOTSTRAPPED_COMMANDS` so the AI can automatically discover and plan this command.

#### [MODIFY] [ai-planning.ts](file:///home/ubuntu/slopware/packages/db/src/services/ai-planning.ts)

- Update the system prompt builder to specialize rules based on `mail-classification` and `mail-to-document-draft` scopes.
- Ensure the AI maps intention classification to one of:
  - `Bestellung / Auftrag`
  - `Angebotsanfrage`
  - `Reklamation`
  - `Liefertermin-/Statusanfrage`
  - `Rechnung / Beleg`
  - `Sonstiges / unklar`
- In `applyPlan`: Add synchronous transactional handling for `apply-ai-mail-classification`. This command updates `emailThread` by setting its `relatedAddressId` and/or `relatedDocumentId` inside the active transaction.

### Component 2: Frontend App (`apps/web`)

#### [MODIFY] [email.tsx](file:///home/ubuntu/slopware/apps/web/src/routes/_auth/app/email.tsx)

- **Toolbar & Commands Integration**:
  - Add an "AI Assistant" button in the email thread details action bar (next to Reply / Forward).
  - Register a new keyboard shortcut command in the `useEffect` command registration through `CommandProvider` (primary shortcut `Alt+A` for "Analyse Thread with AI").
- **Overlay Side Sheet**:
  - Render a sliding side-sheet (`Sheet` from `@repo/ui/components/sheet`) controlled by `isAiSheetOpen: boolean`.
  - When opened, trigger the `POST /api/ai/plan` endpoint for the active `selectedThreadId`.
- **Interactive Action Surface inside Sheet**:
  - Display the thread analysis (intention badge, explanation, confidence meter).
  - List suggested actions checklist:
    1. **Customer Link**: Show unique match or a selectable list of address candidates if ambiguous (allowing user to override/pick the right address contact).
    2. **Document Link**: Show the proposed document draft (e.g., Offer or Order) with listed line items, quantities, and matching articles, allowing the user to select the draft type or override quantities.
    3. **Classification Action**: Command payload mapping to `apply-ai-mail-classification`.
  - Display dynamic backend warnings and dry-run validation statuses.
  - Present an **"Apply Action Plan"** button that runs the plan transactionally via `/api/ai/plans/:planId/apply`.
  - Close sheet and trigger query invalidation on success, displaying a success toast.
