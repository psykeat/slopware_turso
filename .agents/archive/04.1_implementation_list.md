slopware — Module & Component Implementation List
A metadata-driven, keyboard-first B2B ERP shell. Every block below is a discrete, self-contained build unit. Components composed from other components note their dependencies inline.

0. Foundation Layer
   0.1 Design Tokens (tokens.css)
   Single source of truth. Build this first — every other component reads from it.

Color tokens (light mode): --primary, --primary-deep, --primary-soft, --primary-fg, --brand-dark-900, --ink, --ink-secondary, --ink-mute, --canvas, --canvas-soft, --hairline, --hairline-input, --destructive, --ok, --warn
Dark mode overrides via [data-mode="dark"] selector on <html>
10 theme palettes via [data-theme="indigo|ocean|cyan|teal|emerald|forest|amber|rose|violet|slate"] — each provides (primary, primary-deep, primary-soft, primary-fg) for BOTH modes (dark primaries lifted in lightness; primary-fg flips to keep AA contrast on solid surfaces)
Typography tokens: font families (Inter 300 + JetBrains Mono), size/weight/letterspacing for Display 26px, Body 15px, Tabular 14px tnum, Caption 13px, Micro 11px, Mono
Radii: xs 4, sm 6, md 8, lg 12, pill 9999
Shadows: --shadow-card, --shadow-modal, --shadow-kbd
Utility classes: .t-display, .t-body, .t-tabular, .t-caption, .t-micro, .t-mono, .tnum, .upper, .muted, .secondary
Tinted fill rule: all soft fills use color-mix(in oklab, var(--primary) X%, var(--canvas)) — never hard-coded
Focus ring rule: color-mix(in oklab, var(--primary) 22%, transparent)
0.2 Icon System (Icon component)
Single component: <Icon name size={16} stroke={1.5} />
Lucide-style stroke SVGs (24×24 viewBox), currentColor
Needed names: user, package, file, gear, shield, globe, folder, folderOpen, chevR, chevDown, plus, edit, copy, check, archive, send, refresh, box, filter, download, more, search, help, keyboard, x, inbox, truck
All chrome icons go here. Never hand-roll inline SVG elsewhere.
0.3 Tweaks Panel Infrastructure (vendor)
TweaksPanel (host protocol: **edit_mode_available/**activate*edit_mode/**deactivate_edit_mode/**edit_mode_set_keys)
useTweaks(defaults) hook
Tweak controls: TweakSection, TweakSlider, TweakToggle, TweakRadio, TweakSelect, TweakText, TweakNumber, TweakColor (with swatch options), TweakButton
TWEAK_DEFAULTS block wrapped in /\_EDITMODE-BEGIN*/.../_EDITMODE-END_/ markers (valid JSON)

1. Reusable Primitives
   1.1 NavigationTree
   Props: { nodes, selected, onSelect, loading, header }

Reused by: Addresses (Categories), Articles (Article Groups), Documents (Document Types) — and any future hierarchical module.

Node shape: { id, label, level, count, icon?, open? }
28px row, 12px indent per level, chevron + icon + label + count
States: default, hover (canvas bg), active (solid primary bg, white text), loading (~7 indented skeleton rows)
Header strip at top (canvas-soft)
Optional expand/collapse via open prop
1.2 DataGrid
Props: { columns, rows, selected, onSelect, loading, title, count, empty, toolbar }

The single workhorse list view. Reused for: Addresses, Articles, Documents, Admin Users, Related Documents tab, Document Lines tab.

Column shape: { key, label, width, num?, render? }
Toolbar (optional, top): title + count + right-aligned icon buttons (Filter, Refresh, Export, More)
Header row: 36px, canvas-soft bg, caption labels, ink-mute color
Body rows: 40px, ink @ 300, hairline divider, click-to-select
Selected row: color-mix(primary 9%) bg + 2px primary-soft left accent
Numeric columns: right-align + font-variant-numeric: tabular-nums
Custom cell render: render(row) => ReactNode (used for mono IDs, money, status dots)
Loading state: 8 skeleton rows matching column widths
Empty state: { title, subtitle, action: { label, kbd, onClick } } → dashed icon-ring + caption + optional primary "New" pill
toolbar={false} mode for embedded grids (Related Documents, Document Lines)
1.3 ContextTabs
Props: { tabs, active, onChange, children }

Reused by: Address detail, Article detail, Document detail — any record drill-down.

Tab shape: { id, label, count? }
32px tab bar, underline-style
Inactive: ink-mute. Active: ink + 2px primary underline
count renders as a small badge next to label
Right-side icon button slot (e.g. "open in new window")
Body container: .inspector scroll region
1.4 InspectorPanel
Props: { title, id, fields | sections, actions }

Reused inside every ContextTabs detail tab.

40px header (canvas-soft): title + mono ID + right-aligned action icons (default: edit, more)
2-column grid body of (label, value) rows
Field shape: { label, value, mono?, full? } (full spans both columns)
Section shape: { title, fields[] } — sections get a small "section" eyebrow divider
Empty value → em-dash placeholder
1.5 EntityMask
Props: { mode, values, onSubmit, onCancel, focusField }

The create/edit form. Replaces the lower pane inside a TriView when active.

lg-radius card on canvas, 24px padding
Title (Display 26px) + sub-caption ("Press F10 to save…")
2-column form grid; rows declare full to span both columns
Field types reused across all masks: text input, select, checkbox-row
Input states: default (h-9, canvas bg, hairline-input border, sm radius), focused (primary border + 3px primary-ringed shadow), error (destructive border + error-text below)
Required marker: \* after label in destructive color
Help text: micro, ink-mute, below input
Footer: right-aligned Cancel (Esc) + primary Create/Update (F10) pills
focusField prop selects which field gets autofocus
One mask per entity type (Address mask shown; Article/Document/User masks follow same pattern)
1.6 ShortcutOverlay
Props: { open, onClose }

Modal with backdrop, max-w-xl
Header: keyboard icon + "Keyboard Shortcuts" title + sub + close
Body: grouped (label, kbd[]) rows from SHORTCUTS data
Reused regardless of active module

One accent only — the active theme's primary. No secondary accent for interactive elements.
No pure black — use --ink.
No colored status badges — dot + ink-mute label. Color is the only signal.
No gradients, illustrations, or decorative SVG on the work surface.
No padding bloat — 40px max row height (forms excepted).
Tabular numerals for every ID, date, money, and quantity (tnum / font-variant-numeric: tabular-nums).
Mono font for record IDs, codes, kbd badges only.
All recoloring through tokens / color-mix — no theme-specific JS branches.
Extend Icon set rather than inlining new SVG.
No left sidebar. Module nav lives in the AppBar; in-module hierarchy in the TriView NavigationTree.
