# Skill: Triage

## Goal

Manage issue workflow through a state machine driven by specific triage roles.

## Triggers

- Invoked via `/triage`.
- User wants to review or organize issues.

## Core Instructions

- **Assign Categories:** `bug` or `enhancement`.
- **Assign State:**
  - `needs-triage`: New issue.
  - `needs-info`: Waiting for reporter.
  - `ready-for-agent`: Prepared for AFK implementation.
  - `ready-for-human`: Complex, requires human.
  - `wontfix`: Closed with explanation.
- **Process:**
  - **Bugs:** Attempt reproduction before triaging.
  - **Needs Info:** Use a template to clarify knowns/unknowns.
  - **Ready for Agent:** Include a durable agent brief.
- **Signature:** Every triage comment must start with: `> *This was generated by AI during triage.*`
